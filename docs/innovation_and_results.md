# 模型创新点与实验结果分析

## 1. 项目定位

本项目面向边缘部署场景，设计并实现了一个轻量级文本分类模型优化系统。项目并非单纯完成知识蒸馏训练，而是围绕边缘设备对模型体积、推理速度和部署格式的要求，构建了从教师模型训练、学生模型蒸馏、模型评估、ONNX 导出到 INT8 量化的完整流程。

当前第一版实验采用：

| 类型 | 选择 |
| ---- | ---- |
| 数据集 | AG News |
| 教师模型 | `distilbert-base-uncased` |
| 学生模型 | `google/bert_uncased_L-2_H-128_A-2` |
| 蒸馏方式 | Soft Label + 动态温度 + 动态损失权重 + 中间特征对齐 |
| 部署格式 | ONNX |
| 量化方式 | ONNXRuntime INT8 动态量化 |

该系统的核心目标不是单纯追求最高精度，而是在精度、模型体积和推理效率之间取得更适合边缘部署的平衡。

## 2. 与普通知识蒸馏方法的区别

### 2.1 面向边缘部署的端到端优化流程

常见知识蒸馏项目通常只关注教师模型到学生模型的训练过程，实验结果也多以 Accuracy 或 Loss 为主。

本项目进一步将模型优化流程扩展为：

```text
教师模型训练
→ 学生模型知识蒸馏
→ 参数量统计
→ 模型大小统计
→ 推理延迟测试
→ ONNX 导出
→ INT8 动态量化
→ 部署前性能评估
```

因此，本项目的创新点不只体现在蒸馏训练本身，还体现在面向边缘部署的完整工程闭环。

### 2.2 动态温度机制

传统知识蒸馏通常使用固定温度参数，例如在整个训练过程中保持 `T=4`。这种方式无法区分训练前期和后期学生模型的学习需求。

本项目引入动态温度机制：

```text
训练前期：使用较高温度，使学生模型学习教师模型输出分布中的类别间关系
训练后期：逐步降低温度，使学生模型更加关注明确的分类边界
```

可以表述为：

> 本项目设计了面向轻量级学生模型的动态温度调节策略，使学生模型在训练早期更多吸收教师模型的软标签分布信息，在训练后期逐步强化类别判别能力。

### 2.3 动态损失权重

普通知识蒸馏方法通常采用固定损失权重：

```text
Loss = alpha * DistillationLoss + beta * CrossEntropyLoss
```

本项目采用动态损失权重：

```text
训练前期：提高蒸馏损失权重，使学生模型优先学习教师模型的输出分布
训练后期：提高交叉熵损失权重，使学生模型逐步回归真实标签监督
```

这种策略可以降低学生模型对教师错误预测的过度依赖，同时保留教师模型软标签中包含的类别间知识。

### 2.4 中间特征对齐与投影层适配

只进行 logits 蒸馏时，学生模型只能学习教师模型的最终输出分布，无法直接学习教师模型中间层的语义表示。

本项目在 soft label 蒸馏之外，加入中间特征对齐机制：

```text
Student CLS hidden state
→ Linear Projection
→ Teacher hidden size
→ MSE Loss
```

由于教师模型和学生模型隐藏层维度不同，本项目没有直接进行特征 MSE，而是引入线性投影层，将学生模型特征映射到教师模型特征空间后再进行对齐。

可以表述为：

> 针对教师模型与学生模型隐藏层维度不一致的问题，本项目引入特征投影层，将学生模型中间表示映射到教师模型特征空间，并通过 MSE 损失增强学生模型的语义表征能力。

### 2.5 精度、体积、速度三维联合评估

普通知识蒸馏实验往往只报告分类准确率。本项目同时评估：

```text
Accuracy
Precision
Recall
F1 Score
参数量
模型大小
PyTorch 推理延迟
ONNX 推理延迟
INT8 量化后模型大小
量化误差
```

因此，本项目更符合边缘部署场景下对模型实际可用性的评估需求。

### 2.6 蒸馏与 INT8 量化串联

本项目不是单独进行知识蒸馏，也不是单独进行模型量化，而是将两者串联：

```text
知识蒸馏
→ ONNX 导出
→ INT8 动态量化
→ 推理验证
```

这种方式使学生模型不仅在结构上更加轻量，而且在部署格式和存储体积上进一步压缩，更适合边缘设备部署。

## 3. 当前实验结果

当前实验为 Mac 环境下的小样本冒烟测试，主要目的是验证完整流程的可运行性。实验数据量较小，因此结果不能直接作为最终论文中的充分对比实验，但可以证明系统流程已经跑通。

### 3.1 教师模型结果

| 指标 | 结果 |
| ---- | ---- |
| Accuracy | 0.892 |
| Precision | 0.8892 |
| Recall | 0.8871 |
| F1 Score | 0.8871 |

### 3.2 蒸馏学生模型结果

| 指标 | 结果 |
| ---- | ---- |
| Accuracy | 0.772 |
| Precision | 0.7780 |
| Recall | 0.7646 |
| F1 Score | 0.7613 |

### 3.3 模型规模与推理性能

| 指标 | 结果 |
| ---- | ---- |
| 学生模型参数量 | 4,386,436 |
| PyTorch 模型目录大小 | 18.03 MB |
| PyTorch MPS 平均延迟 | 4.42 ms |
| ONNX FP32 模型大小 | 16.78 MB |
| ONNX INT8 模型大小 | 4.28 MB |
| INT8 量化体积减少比例 | 74.52% |
| PyTorch 与 ONNX 最大误差 | 1.19e-7 |

| INT8 ONNX 平均延迟 | 0.283 ms |
| FP32 与 INT8 最大误差 | 0.00958 |

## 4. 关于“是否优于市面模型”的准确表述

当前结果可以证明：

1. 本项目已经完成从训练到部署前优化的完整流程。
2. 蒸馏后的学生模型参数量较小，适合边缘部署。
3. ONNX INT8 量化后，模型体积从 16.78 MB 降低到 4.28 MB，减少约 74.52%。
4. 量化后 logits 误差较小，说明量化没有造成严重输出偏移。

但是，当前实验不能严谨地直接宣称：

```text
本模型确实比市面上所有知识蒸馏模型更好。
```

原因是：

1. 当前只是小样本冒烟测试，不是完整数据集实验。
2. 尚未与 MobileBERT、TinyBERT、MiniLM、DistilBERT 等轻量化模型在同一数据集、同一硬件、同一评估指标下进行公平对比。
3. “市面上的模型”范围过大，不同模型的目标不同，有的追求精度，有的追求速度，有的追求体积。

更稳妥、也更适合论文和答辩的表述是：

> 实验结果表明，本项目构建的轻量化文本分类模型在保持基本分类性能的同时，显著降低了模型体积，并完成了 ONNX 导出与 INT8 动态量化，具备面向边缘部署的实际可行性。与只关注精度的普通知识蒸馏方法相比，本项目更加注重精度、体积和推理效率之间的综合平衡。

如果需要与市面模型比较，可以使用如下表述：

> 相比直接部署通用 BERT 类模型，本项目通过轻量学生模型、知识蒸馏和 INT8 量化显著降低了模型参数量和存储体积，更适合资源受限的边缘设备部署。

在完成全量数据集训练和公平对比实验后，可以进一步表述为：

> 在相同数据集和相同推理环境下，本项目方法在模型体积和部署效率方面相较部分通用预训练模型具有优势，同时保持了可接受的分类性能。

## 5. 可写入论文的创新点总结

建议论文中将创新点总结为以下三点：

### 创新点一：动态蒸馏训练策略

本项目提出并实现了结合动态温度和动态损失权重的知识蒸馏训练策略，使学生模型在训练前期侧重学习教师模型的软标签分布信息，在训练后期逐步增强真实标签监督，从而提高轻量学生模型的训练稳定性。

### 创新点二：基于投影层的中间特征对齐

针对教师模型和学生模型隐藏层维度不一致的问题，本项目引入特征投影层，将学生模型中间表示映射到教师模型特征空间，并通过 MSE 损失进行特征对齐，从而在 logits 蒸馏之外进一步增强学生模型的语义表示学习能力。

### 创新点三：面向边缘部署的轻量化优化流程

本项目构建了知识蒸馏、模型统计、ONNX 导出、INT8 动态量化和推理性能评估的一体化流程，不仅关注模型分类精度，还从参数量、模型体积、推理延迟和量化误差等角度评估模型的边缘部署可行性。
